# Trust & Security (RU)

Этот документ помогает снизить ложные срабатывания антивирусов и повысить доверие к релизам.

## 1) Код-подпись
- Рекомендуется OV/EV code-signing сертификат.
- Подписывать `.exe`/инсталлер на каждом релизе.
- Использовать одинаковое имя издателя.

## 2) Прозрачность релиза
- Публиковать SHA256 для всех артефактов.
- Публиковать changelog и список изменений.
- Держать README и документацию актуальными.

## 3) Стабильность сборки
- Зафиксированные версии зависимостей.
- Без обфускации/упаковщиков, похожих на malware-паттерны.
- Минимум лишних артефактов в Assets.

## 4) Ложные срабатывания
- Подача false-positive в Microsoft Defender и другие AV.
- Добавлять ссылки на отчёты VirusTotal в релизы (по желанию).

## 5) Практика для пользователей
- Скачивать только из официального GitHub Releases.
- Сверять SHA256 перед запуском.
